OPPO Credentialed CORS Exploit Proof of Concept

This Proof of Concept demonstrates a credentialed cross-origin data exposure via misconfigured CORS policy on multiple authenticated API endpoints. The exploit uses XMLHttpRequest with credentials enabled, allowing session cookies to be included automatically and sensitive account data to be retrieved cross-origin. Click the button below while logged into an OPPO account to reproduce the issue.
Affected endpoints: /cart/v1/list /cart/getCount /coupons/count /credits/getUserCredit /credits/show
